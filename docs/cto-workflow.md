# CTO Workflow

Use the brain to separate architecture intent from implementation truth.

## Weekly Review

- Check `04-Engineering`, `05-Production`, `06-Security`, and `15-Decisions-ADR`.
- Verify important claims against GitHub, tests, and runtime systems.
- Update the source-of-truth map when repos, services, or branches change.
- Convert architectural changes into ADRs.

## Before A Technical PR

- Create a narrow task packet.
- State contracts, files likely to change, tests, and risk.
- Tell agents which context pack to read.
- Require evidence in the PR body.
